  2. 点击任意告警的"屏蔽"按钮
  3. ✅ 验证弹窗显示三元组说明
  4. ✅ 验证显示"屏蔽类型"选择（固定时长/周期性）
  5. 选择"固定时长" -> "2小时"

  7. 点击"确定"
  8. ✅ 验证告警显示"屏蔽中"标签
测试2: 标签编辑功能
  1. 点击告警的"屏蔽"按钮

  7. 选择"固定时长" -> "1小时"
  8. 点击"确定"
  9. ✅ 验证所有相同 job 和 instance 的告警都被屏蔽
测试3: 周期性屏蔽

  7. 设置结束时间：18:00

⚠️ 测试清单 - 批量选择功能（重点）

⚠️ 重要：这是用户反馈的主要问题，请仔细测试每一步
测试4: 批量选择基础功能
  1. 进入"实时告警"页面
  2. 🔍 检查点1: 表格第一列是否显示 checkbox 列？
    • ✅ 如果显示：继续下一步
    • ❌ 如果不显示：打开浏览器控制台，查看是否有错误
  3. 🔍 检查点2: 点击表头的 checkbox（全选）
    • ✅ 如果所有行都被选中：功能正常
    • ❌ 如果无反应：记录问题
  4. 🔍 检查点3: 点击单个告警行的 checkbox
    • ✅ 如果该行被选中：功能正常
    • ❌ 如果无反应：记录问题
  5. 🔍 检查点4: 选中2-3条告警后，批量操作栏是否显示？
    • ✅ 如果显示蓝色操作栏，显示"已选 X 条"：功能正常
    • ❌ 如果不显示：记录问题
测试5: 批量屏蔽功能
  1. 勾选2-3条告警
  2. 点击批量操作栏中的"批量屏蔽"按钮
  3. ✅ 验证弹窗显示完整功能（三元组说明、标签编辑、类型选择）
  4. 选择"固定时长" -> "2小时"
  5. 输入原因："批量测试"
  6. 点击"确定"
  7. ✅ 验证所有选中的告警都显示"屏蔽中"
  8. ✅ 验证批量操作栏消失
  9. ✅ 验证选中状态被清空
测试6: 已屏蔽告警批量取消
  1. 点击"已屏蔽告警"按钮
  2. ✅ 验证弹窗显示已屏蔽的告警列表
  3. 🔍 检查点5: 表格第一列是否显示 checkbox？
  4. 勾选2-3条已屏蔽告警
  5. 🔍 检查点6: 批量操作栏是否显示？
  6. 点击"批量取消屏蔽"
  7. 确认操作
  8. ✅ 验证告警从列表中移除
  9. 关闭弹窗，返回实时告警列表
  10. ✅ 验证告警恢复正常（不再显示"屏蔽中"）

🐛 如果批量选择不工作

请收集以下信息：
1. 浏览器控制台错误
  • 按 F12 打开开发者工具
  • 切换到 Console 标签
  • 刷新页面，查看是否有红色错误信息
  • 截图或复制错误信息
2. 检查数据格式

在浏览器控制台执行：

// 检查告警数据 console.log('events:', events.value) console.log('first event:', events.value[0]) console.log('first event id:', events.value[0]?.id) // 检查选中状态 console.log('selectedEventIds:', selectedEventIds.value) console.log('rowSelection:', rowSelection.value)
3. 检查 Arco Design 版本
cd /Users/Zhuanz/golang_project/src/opshub/web npm list @arco-design/web-vue
4. 视觉检查
  • 表格第一列是否有空白列（checkbox 列）？
  • 鼠标悬停在 checkbox 位置时，光标是否变成手型？
  • 点击 checkbox 时是否有任何视觉反馈？
