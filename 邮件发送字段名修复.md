# 邮件发送字段名修复

## 问题

邮件发送时报错：
```
Error 1054 (42S22): Unknown column 'name' in 'field list'
```

## 原因

代码中使用了 `name` 字段，但 `sys_user` 表中的字段名是 `real_name`。

## 修复

**文件**：`internal/service/alert/notify_service.go`

**修改前**：
```go
type userRow struct {
    Email string
    Name  string  // ❌ 错误：表中没有 name 字段
}
s.db.WithContext(ctx).Table("sys_user").
    Select("email, name").  // ❌ 错误
    Where("phone IN ? AND email != ''", phones).
    Scan(&users)
```

**修改后**：
```go
type userRow struct {
    Email    string
    RealName string  // ✅ 正确：使用 real_name
}
s.db.WithContext(ctx).Table("sys_user").
    Select("email, real_name").  // ✅ 正确
    Where("phone IN ? AND email != ''", phones).
    Scan(&users)
```

## sys_user 表结构

```sql
CREATE TABLE `sys_user` (
  `id` bigint unsigned NOT NULL AUTO_INCREMENT,
  `username` varchar(50) NOT NULL COMMENT '用户名',
  `password` varchar(255) NOT NULL COMMENT '密码',
  `real_name` varchar(50) DEFAULT NULL COMMENT '真实姓名',  -- ✅ 字段名是 real_name
  `email` varchar(100) DEFAULT NULL COMMENT '邮箱',
  `phone` varchar(20) DEFAULT NULL COMMENT '手机号',
  `avatar` varchar(255) DEFAULT NULL COMMENT '头像',
  `status` tinyint DEFAULT '1' COMMENT '状态 1:启用 0:禁用',
  `department_id` bigint unsigned DEFAULT '0' COMMENT '部门ID',
  `bio` text COMMENT '个人简介',
  `last_login_at` datetime(3) DEFAULT NULL COMMENT '最后登录时间',
  `created_at` datetime(3) DEFAULT NULL,
  `updated_at` datetime(3) DEFAULT NULL,
  `deleted_at` datetime(3) DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
```
