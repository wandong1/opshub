# 告警订阅邮件通道修复说明

## 问题描述

用户在告警订阅中选择了邮件通道和接收用户，但没有收到邮件通知。

## 问题原因

经过排查，发现以下问题：

1. **`notify_service.go` 的 `Send` 方法缺少 `email` 类型处理**
   - 文件：`internal/service/alert/notify_service.go`
   - 位置：第 72-88 行的 switch 语句
   - 问题：只处理了 `wechat_work`、`dingtalk`、`sms`、`phone`、`ai_agent`，没有 `email`

2. **缺少邮件发送实现**
   - 告警治理模块的邮件发送逻辑未实现
   - 需要查询用户邮箱地址（通过手机号关联）

3. **架构问题**
   - `NotifyService` 缺少数据库连接，无法查询用户邮箱

### 3. ✅ 实现邮件发送逻辑

**文件**：`internal/service/alert/notify_service.go`

**新增方法**：
- `sendEmail(ctx, configJSON, msg, phones)` - 主发送逻辑
- `sendEmailWithTLS(addr, host, user, password, from, to, msg)` - 端口 465
- `sendEmailWithSTARTTLS(addr, host, user, password, from, to, msg)` - 端口 587

**核心逻辑**：
```go
func (s *NotifyService) sendEmail(ctx context.Context, configJSON, msg string, phones []string) error {
	// 1. 解析邮件配置
	var cfg struct {
		SMTPHost     string `json:"smtpHost"`
		SMTPPort     int    `json:"smtpPort"`
		SMTPUser     string `json:"smtpUser"`
		SMTPPassword string `json:"smtpPassword"`
		FromEmail    string `json:"fromEmail"`
		FromName     string `json:"fromName"`
	}
	
	// 2. 检查是否有接收用户
	if phones == nil || len(phones) == 0 {
		appLogger.Info("邮件通道未指定接收用户，跳过发送")
		return nil
	}
	
	// 3. 通过手机号查询用户邮箱
	var users []struct {
		Email string
		Name  string
	}
	s.db.WithContext(ctx).Table("sys_user").
		Select("email, name").
		Where("phone IN ? AND email != ''", phones).
		Scan(&users)
	
	// 4. 构建邮件内容
	subject := "SreHub 告警通知"
	headers := map[string]string{
		"From":         fmt.Sprintf("%s <%s>", cfg.FromName, cfg.FromEmail),
		"To":           strings.Join(emails, ", "),
		"Subject":      subject,
		"MIME-Version": "1.0",
		"Content-Type": "text/html; charset=UTF-8",
	}
	
	// 5. 根据端口选择发送方式
	if cfg.SMTPPort == 465 {
		err = sendEmailWithTLS(...)  // 直接 TLS
	} else {
		err = sendEmailWithSTARTTLS(...)  // STARTTLS
	}
	
	return err
}
```

## 邮件发送逻辑说明

### 用户选择逻辑

1. **选择了接收用户**（`phones` 非空）：
   - 通过手机号查询 `sys_user` 表的 `email` 字段
   - 只发送给有邮箱的用户
   - 如果没有找到有效邮箱，返回错误

2. **未选择接收用户**（`phones` 为空或 nil）：
   - 跳过邮件发送
   - 记录日志：`邮件通道未指定接收用户，跳过发送`

### 邮件内容

- **主题**：
  - 告警：`SreHub 告警通知`
  - 恢复：`SreHub 恢复通知`（检测消息中是否包含"恢复"或"✅"）

- **内容格式**：HTML（支持用户自定义的 HTML 模板）

- **发件人**：`{FromName} <{FromEmail}>`

### TLS 加密

- **端口 465**：使用 `sendEmailWithTLS`（直接 TLS 连接）
- **端口 587/25**：使用 `sendEmailWithSTARTTLS`（STARTTLS 升级）

### 日志记录

所有关键步骤都有详细日志：
```
INFO    准备发送邮件    {"smtpHost": "smtp.gmail.com", "smtpPort": 465, "phones": ["13800138000"]}
INFO    找到接收邮箱    {"emails": ["user@example.com"]}
INFO    邮件发送成功    {"emails": ["user@example.com"]}
```

或错误日志：
```
ERROR   查询用户邮箱失败    {"error": "..."}
WARN    未找到有效的用户邮箱    {"phones": ["13800138000"]}
ERROR   邮件发送失败    {"smtpHost": "smtp.gmail.com", "error": "..."}
```

## 使用说明

### 1. 配置邮件通道

进入"告警治理" → "告警通道管理" → "新增通道"：

```json
{
  "name": "Gmail邮件通知",
  "type": "email",
  "enabled": true,
  "config": {
    "smtpHost": "smtp.gmail.com",
    "smtpPort": 465,
    "smtpUser": "your-email@gmail.com",
    "smtpPassword": "your-app-password",
    "fromEmail": "your-email@gmail.com",
    "fromName": "OpsHub告警"
  },
  "alertTemplate": "HTML模板内容",
  "resolveTemplate": "HTML模板内容"
}
```

### 2. 配置告警订阅

进入"告警治理" → "告警订阅" → "新增订阅"：

1. 选择资产分组
2. 选择告警规则
3. **选择通知通道**：勾选邮件通道
4. **选择接收用户**：勾选需要接收邮件的用户
5. 保存

### 3. 用户邮箱配置

**重要**：接收用户必须在系统中配置邮箱地址。

进入"系统管理" → "用户管理" → 编辑用户：
- 确保用户的"邮箱"字段已填写
- 系统通过用户的手机号查询邮箱地址

### 4. 查看日志

```bash
# 查看邮件发送日志
tail -f logs/app.log | grep "邮件"

# 查看告警通知日志
tail -f logs/app.log | grep "告警通知"
```

## 注意事项

### 1. 用户邮箱必填

- 系统通过 `sys_user` 表的 `phone` 字段查询用户
- 然后获取对应的 `email` 字段
- 如果用户没有邮箱，会跳过该用户

### 2. 邮件配置

- **Gmail**：需要使用"应用专用密码"
- **QQ 邮箱**：需要开启 SMTP 服务并使用授权码
- **163 邮箱**：需要开启 SMTP 服务并使用授权码

### 3. 端口选择

- **端口 465**：直接 TLS（推荐）
- **端口 587**：STARTTLS
- **端口 25**：不推荐（不安全）

### 4. 未选择用户的行为

如果告警订阅中**未选择接收用户**：
- 邮件通道会跳过发送
- 其他通道（企业微信、钉钉）会 @all

这是合理的设计，因为：
- 邮件需要明确的收件人地址
- 不能像 IM 工具那样 @all

### 5. HTML 模板支持

邮件内容支持 HTML 格式，可以使用：
- 自定义 HTML 模板（参考 `告警邮件HTML模板.md`）
- 模板变量：`{{.RuleName}}`、`{{.SeverityLabel}}` 等

## 排查问题

### 问题 1：没有收到邮件

**检查步骤**：
1. 查看日志是否有"邮件发送成功"
2. 检查用户是否配置了邮箱
3. 检查 SMTP 配置是否正确
4. 检查邮箱垃圾箱

### 问题 2：日志显示"未找到有效的用户邮箱"

**原因**：选择的用户没有配置邮箱地址

**解决**：
1. 进入"系统管理" → "用户管理"
2. 编辑用户，填写邮箱地址
3. 重新触发告警

### 问题 3：日志显示"邮件通道未指定接收用户"

**原因**：告警订阅中未选择接收用户

**解决**：
1. 进入"告警治理" → "告警订阅"
2. 编辑订阅，选择接收用户
3. 保存

### 问题 4：SMTP 认证失败

**原因**：密码错误或未使用授权码

**解决**：
- Gmail：使用"应用专用密码"，不是登录密码
- QQ/163：使用"授权码"，不是邮箱密码
