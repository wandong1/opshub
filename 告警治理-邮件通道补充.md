## 问题说明

用户反馈"告警通道管理"页面没有邮件选项，经检查发现：

1. **用户打开的是告警治理模块**（`web/src/views/alert/Channels.vue`）
2. **不是监控插件的通道管理**（`web/src/plugins/monitor/components/AlertChannels.vue`）

## 使用说明

### 1. 新增邮件通道

1. 进入"告警治理" → "告警通道管理"
2. 点击"新增通道"
3. 选择通道类型：**邮件**
4. 填写 SMTP 配置：
   - SMTP服务器：如 `smtp.gmail.com`
   - SMTP端口：465（TLS）或 587（STARTTLS）
   - 发件人邮箱：如 `noreply@example.com`
   - 发件人名称：如 `OpsHub告警`
   - SMTP用户名：通常与发件人邮箱相同
   - SMTP密码：邮箱授权码（不是登录密码）
5. 点击"确定"保存

### 2. 测试邮件通道

1. 在通道列表中找到刚创建的邮件通道
2. 点击"测试"按钮
3. 查看是否收到测试邮件

### 3. 配置告警规则使用邮件通道

1. 进入"告警治理" → "告警规则"
2. 编辑或新增规则
3. 在"通知通道"中选择邮件通道
4. 保存规则

## 注意事项

1. **Gmail**：
   - 需要开启"两步验证"
   - 使用"应用专用密码"，不是账号密码
   - 生成方式：Google账号 → 安全性 → 两步验证 → 应用专用密码

2. **QQ 邮箱**：
   - 需要开启 SMTP 服务
   - 使用"授权码"，不是 QQ 密码
   - 获取方式：QQ邮箱 → 设置 → 账户 → POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务

3. **163 邮箱**：
   - 需要开启 SMTP 服务
   - 使用"授权码"，不是邮箱密码
   - 获取方式：163邮箱 → 设置 → POP3/SMTP/IMAP → 开启服务

4. **端口选择**：
   - 端口 465：使用直接 TLS 加密（推荐）
   - 端口 587：使用 STARTTLS 升级加密
   - 端口 25：不推荐（不安全）

5. **发送失败排查**：
   - 查看日志：`tail -f logs/app.log | grep "邮件"`
   - 检查 SMTP 配置是否正确
   - 确认使用的是授权码而不是登录密码
   - 检查网络是否能访问 SMTP 服务器

## 两个模块的区别

| 功能 | 告警治理模块 | 监控插件 |
|------|------------|---------|
| 路径 | `/alert/channels` | `/plugins/monitor/alerts/channels` |
| 文件 | `web/src/views/alert/Channels.vue` | `web/src/plugins/monitor/components/AlertChannels.vue` |
| 用途 | Prometheus 告警规则通知 | 域名监控告警通知 |
| 支持通道 | 邮件、企业微信、钉钉、短信、电话、AI智能体 | 邮件、Webhook、企业微信、钉钉、飞书 |
| 后端 API | `/api/v1/alert/channels` | `/api/v1/plugins/monitor/alerts/channels` |
