# 告警订阅快速启用/禁用功能修复

## 🐛 问题描述

**问题**：
- 用户在列表页点击"启用"开关关闭订阅时
- 订阅的推送规则配置被清空
- 重新启用后，需要重新配置推送规则

**影响**：
- 用户无法临时关闭订阅
- 关闭后配置丢失
- 用户体验极差

---

## 🔍 问题原因

### 前端逻辑分析

**列表页的启用开关**（第 20 行）：
```vue
<a-switch :model-value="record.enabled" size="small" @change="(v) => quickToggle(record, !!v)" />
```

**quickToggle 方法**（第 329-331 行）：
```typescript
const quickToggle = async (row: any, v: boolean) => {
  await updateSubscription(row.id, { ...row, enabled: v }).catch(() => {}); load()
}
```

**问题**：
1. `row` 对象是列表数据，**只包含基本字段**：
   ```typescript
   {
     id: 5,
     name: "测试订阅",
     enabled: true,
     ruleCount: 1,
     channelCount: 1,
     // ❌ 没有 rules、channels、users 等详细配置
   }
   ```

2. `updateSubscription` 调用完整的更新接口，传递了 `{ ...row, enabled: v }`

3. 后端收到的数据：
   ```json
   {
     "id": 5,
     "name": "测试订阅",
     "enabled": false,
     "rules": [],      // ❌ 空数组
     "channelIds": [], // ❌ 空数组
     "userIds": []     // ❌ 空数组
   }
   ```

4. 后端执行 `saveSubRelations`，将空数组保存到数据库，**清空了原有配置**

### 根本原因

**列表页的启用开关和编辑窗口的启用开关，传输数据的逻辑不一致**：
- **编辑窗口**：有完整的 `rules`、`channels`、`users` 数据
- **列表页**：只有基本字段，没有详细配置

---

## ✅ 修复方案

### 方案：添加专门的 toggle 接口

**核心思路**：
- 添加一个专门的 PATCH 接口 `/api/v1/alert/subscriptions/:id/toggle`
- 只更新 `enabled` 字段
- 不触发完整的校验逻辑
- 不修改其他字段（rules、channels、users）

---

## 📝 修复内容

### 1. 后端：添加 toggle 接口

**文件**：`internal/server/alert/subscription_handler.go`

**新增方法**：
```go
// toggleSubscription 切换订阅启用状态（只更新 enabled 字段）
func (s *HTTPServer) toggleSubscription(c *gin.Context) {
	id, _ := strconv.ParseUint(c.Param("id"), 10, 64)

	// 解析请求体
	var req struct {
		Enabled bool `json:"enabled"`
	}
	if err := c.ShouldBindJSON(&req); err != nil {
		response.ErrorCode(c, http.StatusBadRequest, err.Error())
		return
	}

	// 查询订阅
	sub, err := s.subRepo.GetByID(c.Request.Context(), uint(id))
	if err != nil {
		response.ErrorCode(c, http.StatusNotFound, "订阅任务不存在")
		return
	}

	// 只更新 enabled 字段
	sub.Enabled = req.Enabled
	if err := s.subRepo.Update(c.Request.Context(), sub); err != nil {
		response.ErrorCode(c, http.StatusInternalServerError, "更新失败")
		return
	}

	response.Success(c, sub)
}
```

**特点**：
- ✅ 只更新 `enabled` 字段
- ✅ 不触发 `validateSubscriptionRequest` 校验
- ✅ 不调用 `saveSubRelations`
- ✅ 不修改 `rules`、`channels`、`users`

**文件**：`web/src/views/alert/Subscriptions.vue`

**修改前**：
```typescript
const quickToggle = async (row: any, v: boolean) => {
  await updateSubscription(row.id, { ...row, enabled: v }).catch(() => {}); load()
}
```

**修改后**：
```typescript
const quickToggle = async (row: any, v: boolean) => {
  try {
    await toggleSubscription(row.id, v)
    Message.success(v ? '已启用' : '已禁用')
    load()
  } catch (error) {
    Message.error('操作失败')
  }
}
```

**改进**：
- ✅ 调用专门的 `toggleSubscription` API
- ✅ 只传递 `id` 和 `enabled`
- ✅ 添加成功/失败提示
- ✅ 添加错误处理
