# 邮件发送 @all 问题修复

## 🚨 严重问题

**问题描述**：
- 用户在告警订阅中指定了单个用户
- 但邮件却发送给了所有用户
- 这是非常危险的行为

## 🔍 问题原因

### 1. phones 参数的含义

在告警通知系统中，`phones` 参数有三种状态：

| 状态 | 含义 | IM 工具行为 | 邮件行为（修复前） | 邮件行为（修复后） |
|------|------|------------|------------------|------------------|
| `nil` | @all（所有人） | @all | ❌ 发给所有用户 | ✅ 跳过发送 |
| `[]` | 未选择用户 | 不 @ | ✅ 跳过发送 | ✅ 跳过发送 |
| `["phone1", "phone2"]` | 指定用户 | @指定用户 | ✅ 发给指定用户 | ✅ 发给指定用户 |

### 2. 问题代码

**修复前**（`notify_service.go`）：
```go
// 如果没有选择接收用户，则不发送邮件
if phones == nil || len(phones) == 0 {
    appLogger.Info("邮件通道未指定接收用户，跳过发送")
    return nil
}
```

**问题**：
- `phones == nil` 表示 @all（IM 工具的概念）
- 但代码将 `nil` 和 `[]` 混在一起处理
- 导致 `phones == nil` 时也跳过，但实际上可能已经查询了所有用户

### 3. 根本原因

在 `eval_service.go` 中：

```go
// getUserPhonesByIDs 返回 nil 表示 @all
func (e *EvalEngine) getUserPhonesByIDs(ctx context.Context, userIDs []uint) []string {
    // 检查是否包含 ID=0（@all）
    for _, uid := range userIDs {
        if uid == 0 {
            return nil // nil=@all
        }
    }
    // 查询指定用户的手机号
    // ...
}
```

当用户选择了 `userID=0`（表示所有人）时，返回 `nil`。

## ✅ 修复方案

### 修改邮件发送逻辑

**文件**：`internal/service/alert/notify_service.go`

**修复后**：
```go
// sendEmail 发送邮件通知
func (s *NotifyService) sendEmail(ctx context.Context, configJSON, msg string, phones []string) error {
    // ...

    // 邮件发送必须指定接收用户，不支持 @all
    // phones == nil 表示 @all（IM 工具），但邮件不支持，直接跳过
    // phones == [] 表示未选择用户，也跳过
    if phones == nil {
        appLogger.Info("邮件通道不支持 @all，跳过发送")
        return nil
    }

    if len(phones) == 0 {
        appLogger.Info("邮件通道未指定接收用户，跳过发送")
        return nil
    }

    // 继续发送邮件给指定用户
    // ...
}
```

### 关键改动

1. **分开判断 `nil` 和 `[]`**：
   - `phones == nil` → 记录日志"邮件通道不支持 @all"，跳过发送
   - `len(phones) == 0` → 记录日志"未指定接收用户"，跳过发送

2. **明确语义**：
   - 邮件永远不支持 @all
   - 邮件必须指定明确的收件人

## 🎯 使用建议

### 1. 告警订阅配置

**推荐做法**：
- 明确选择需要接收邮件的用户
- 不要选择"所有人"（userID=0）
- 每个订阅任务指定具体的接收人

**不推荐做法**：
- ❌ 选择"所有人"接收邮件
- ❌ 不选择接收用户

### 2. 邮件 vs IM 工具的区别

| 特性 | 邮件 | 企业微信/钉钉 |
|------|------|--------------|
| @all 支持 | ❌ 不支持 | ✅ 支持 |
| 必须指定用户 | ✅ 是 | ❌ 否（可以 @all） |
| 收件人可见 | ✅ 是 | ❌ 否（群消息） |
| 适用场景 | 重要告警、个人通知 | 团队协作、群通知 |

### 3. 最佳实践

**场景 1：重要告警**
- 使用邮件通道
- 指定具体的负责人
- 确保每个人都能收到

**场景 2：团队通知**
- 使用企业微信/钉钉
- 可以 @all 或 @指定人
- 适合快速响应

**场景 3：混合使用**
- 邮件：发给具体负责人
- IM：@all 通知团队
- 两个通道独立配置

## 🔒 安全性

### 修复前的风险

1. **隐私泄露**：
   - 所有用户都能收到邮件
   - 可能包含敏感信息

2. **邮件轰炸**：
   - 大量用户收到不相关的邮件
   - 影响用户体验

3. **SMTP 限制**：
   - 一次发送给所有用户
   - 可能触发 SMTP 服务器限制

### 修复后的保护

1. **明确收件人**：
   - 只发给指定用户
   - 不支持 @all

2. **日志记录**：
   - 记录所有发送行为
   - 可追溯

3. **错误提示**：
   - 明确告知不支持 @all
   - 引导用户正确配置
