# 告警巡检功能 - 实施指南（基于推送规则）

## 📋 功能概述

告警巡检功能允许为每个**推送规则**独立配置定期巡检策略。系统会按照推送规则的过滤条件（时间范围、告警级别、数据源、标签等）自动巡查符合条件的告警，并生成美观的巡检报告推送给该规则配置的用户。

**核心特点：**
- ✅ **推送规则级别配置**：每个推送规则可以独立配置巡检策略
- ✅ **完全复用过滤规则**：巡检自动应用推送规则的所有过滤条件
- ✅ **灵活的巡检时间**：支持间隔模式和固定时间模式
- ✅ **美观的报告格式**：支持详细报告和摘要报告两种样式

## 🎯 使用指南

### 1. 配置巡检

#### 步骤1：创建或编辑订阅
1. 进入告警订阅管理页面
2. 点击"新增订阅"或"编辑"已有订阅
3. 在"推送规则配置"部分，配置推送规则：
   - 选择触发规则（留空=全部规则）
   - 选择告警级别（留空=全部级别）
   - 配置数据源过滤
   - 配置标签匹配器
   - 配置生效时间
   - 选择通知通道和接收用户

#### 步骤2：配置该推送规则的巡检
1. 点击推送规则右上角的"巡检配置"按钮
2. 配置巡检参数：

**基础配置：**
- **启用巡检**：开启/关闭该推送规则的巡检功能
- **推送模式**：
  - `总是推送`：无论有无告警都推送报告
  - `仅有告警时推送`：只在有活跃告警时推送

**巡检时间配置：**
- **巡检模式**：
  - `按间隔巡检`：每隔N秒执行一次（如每小时=3600秒）
  - `固定时间巡检`：在指定时间点执行（如 09:00、18:00）

**巡检范围配置：**
- **时间范围**：
  - `0`：巡检所有未恢复的告警
  - `>0`：只巡检最近N秒内的告警（如 86400=最近24小时）
- **最大告警数**：单次报告最多包含的告警数量（默认100）
- **包含已恢复**：是否在报告中统计已恢复的告警

**报告样式配置：**
- **报告样式**：
  - `详细报告`：显示每条告警的完整信息（最多20条）
  - `摘要报告`：只显示统计数据和Top告警（最多10条）
- **分组维度**：
  - `按告警级别分组`：critical、warning、info
  - `按规则名称分组`：相同规则的告警合并统计
  - `按业务分组`：按 asset_group 标签分组

### 2. 手动执行巡检

在巡检配置弹窗中点击"手动执行巡检"按钮，可以立即触发一次巡检任务，用于测试配置是否正确。

### 3. 查看巡检报告

在巡检配置弹窗中点击"查看巡检报告"按钮，可以查看该推送规则的历史巡检报告列表。

## 🔧 常见使用场景

### 场景1：每日早报（按业务分组）
**需求**：每天早上9点推送生产环境未解决告警汇总

**配置：**
1. 创建订阅"生产环境告警"
2. 添加推送规则：
   - 触发规则：留空（全部规则）
   - 告警级别：留空（全部级别）
   - 数据源过滤：选择生产环境数据源
   - 标签匹配器：`env = production`
   - 通知通道：飞书群
   - 接收用户：运维团队
3. 配置巡检：
   - 巡检模式：`固定时间巡检`
   - 巡检时间点：`["09:00"]`
   - 时间范围：`86400`（24小时）
   - 推送模式：`总是推送`
   - 报告样式：`详细报告`
   - 分组维度：`按业务分组`

### 场景2：高危告警监控
**需求**：每小时巡检严重级别告警

**配置：**
1. 创建订阅"高危告警监控"
2. 添加推送规则：
   - 触发规则：留空（全部规则）
   - 告警级别：只选择 `critical`
   - 通知通道：短信 + 电话
   - 接收用户：值班人员
3. 配置巡检：
   - 巡检模式：`按间隔巡检`
   - 巡检间隔：`3600`（1小时）
   - 时间范围：`0`（所有未恢复）
   - 推送模式：`仅有告警时推送`
   - 报告样式：`摘要报告`

### 场景3：分级巡检（不同级别不同频率）
**需求**：严重告警每小时巡检，一般告警每4小时巡检

**配置：**
1. 创建订阅"分级巡检"
2. 添加推送规则1（严重告警）：
   - 告警级别：`critical`
   - 通知通道：飞书 + 短信
   - 巡检间隔：`3600`（1小时）
3. 添加推送规则2（一般告警）：
   - 告警级别：`warning`, `info`
   - 通知通道：飞书
   - 巡检间隔：`14400`（4小时）

### 场景4：特定规则巡检
**需求**：只巡检数据库相关的告警

**配置：**
1. 创建订阅"数据库告警巡检"
2. 添加推送规则：
   - 触发规则：选择所有数据库相关的规则
   - 标签匹配器：`component = database`
   - 通知通道：钉钉
   - 接收用户：DBA团队
3. 配置巡检：
   - 巡检模式：`固定时间巡检`
   - 巡检时间点：`["09:00", "18:00"]`
   - 报告样式：`详细报告`

## 🎨 技术特性

### 1. 推送规则级别配置
- 每个推送规则可以独立配置巡检策略
- 不同推送规则可以有不同的巡检频率和报告样式
- 巡检自动应用推送规则的所有过滤条件

### 2. 智能过滤
- 完全遵循推送规则的过滤逻辑
- 支持时间范围、告警级别、数据源、标签匹配器等多维度过滤
- 支持生效时间配置（只在指定时间段内巡检）

### 3. 灵活调度
- 支持间隔模式和固定时间模式
- 自动计算下次巡检时间
- 30秒检查一次，精确执行

### 4. 美观报告
- 使用 emoji 和分隔线美化报告
- 支持详细报告和摘要报告两种样式
- 自动格式化持续时长（如"2天3小时"）

### 5. 多通道推送
- 使用推送规则配置的通道和用户
- 支持飞书、钉钉、企业微信、短信、邮件等

## ⚠️ 注意事项

### 1. 推送规则必须先保存
- 巡检配置依赖推送规则的ID
- 必须先保存订阅，确保推送规则有ID后才能配置巡检
- 如果点击"巡检配置"提示"请先保存订阅"，说明订阅还未保存

### 2. 巡检间隔建议
- 最小间隔：60秒（1分钟）
- 推荐间隔：3600秒（1小时）或更长
- 过短的间隔可能导致频繁推送，影响用户体验

### 3. 时间范围设置
- `0`：巡检所有未恢复的告警（推荐用于日常巡检）
- `>0`：只巡检最近N秒内的告警（推荐用于周报、月报）

### 4. 报告样式选择
- **详细报告**：适合告警数量较少（<20条）的场景
- **摘要报告**：适合告警数量较多的场景，避免报告过长

### 5. 推送模式选择
- **总是推送**：适合需要定期确认系统状态的场景（如早报）
- **仅有告警时推送**：适合只关注异常情况的场景（如高危告警监控）

## 🐛 故障排查

### 问题1：点击"巡检配置"提示"请先保存订阅"
**原因**：推送规则还没有ID
**解决**：先点击"确定"保存订阅，然后重新编辑订阅，再点击"巡检配置"

### 问题2：巡检没有执行
**检查：**
1. 巡检配置是否启用（`enabled = true`）
2. 查看服务日志是否有"告警巡检服务已启动"
3. 检查数据库表 `alert_subscription_rule_patrol` 中的 `next_patrol_at` 字段

### 问题3：没有收到巡检报告
**检查：**
1. 推送模式是否为"仅有告警时推送"，但实际没有告警
2. 推送规则的通道是否启用
3. 推送规则的用户是否配置正确
4. 查看 `alert_patrol_report` 表中的 `sent` 和 `send_error` 字段

### 问题4：巡检报告中没有告警
**检查：**
1. 推送规则的过滤条件是否过于严格
2. 时间范围设置是否合理
3. 推送规则的生效时间是否包含当前时间
4. 手动执行巡检，查看是否有告警

## 📚 API 文档

### 1. 获取推送规则的巡检配置
```
GET /api/v1/alert/patrol/rule/:ruleId
```

### 2. 保存推送规则的巡检配置
```
POST /api/v1/alert/patrol/rule
```

**请求体：**
```json
{
  "subscriptionRuleId": 1,
  "enabled": true,
  "patrolMode": "interval",
  "patrolInterval": 3600,
  "patrolTimes": "[]",
  "includeResolved": false,
  "timeRange": 0,
  "maxAlertsPerReport": 100,
  "sendMode": "always",
  "reportStyle": "detailed",
  "groupBy": "severity"
}
```

### 3. 获取推送规则的巡检报告列表
```
GET /api/v1/alert/patrol/rule/:ruleId/reports?page=1&page_size=20
```

### 4. 手动执行推送规则的巡检
```
POST /api/v1/alert/patrol/rule/:ruleId/execute
```

## 🔄 与订阅级别巡检的对比

| 特性 | 推送规则级别（当前方案） | 订阅级别（旧方案） |
|------|------------------------|------------------|
| 配置粒度 | 每个推送规则独立配置 | 整个订阅统一配置 |
| 过滤规则 | 自动应用推送规则的过滤条件 | 需要重新配置过滤条件 |
| 灵活性 | 高（不同规则不同频率） | 低（所有规则统一频率） |
| 配置复杂度 | 简单（复用推送规则配置） | 复杂（需要重复配置） |
| 使用场景 | 分级巡检、特定规则巡检 | 全局巡检 |

**推荐使用推送规则级别的巡检配置！**
