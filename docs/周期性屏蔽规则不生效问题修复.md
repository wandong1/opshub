# 周期性屏蔽规则不生效问题修复

## 🐛 问题描述

**现象**：
- 固定时长屏蔽：✅ 正常工作
- 周期性屏蔽：❌ 不生效，告警仍然产生并推送

**示例**：
- 屏蔽规则：`主机-内存使用率过高`，周期性屏蔽，每天 09:00-20:00
- 告警时间：2026-05-07 19:03（在屏蔽时间段内）
- **问题**：告警仍然产生并推送通知

---

## 🔍 问题根因

### 问题定位

经过分析，问题出在**前端时间选择器的数据格式**上。

**前端代码**（修复前）：
```vue
<a-time-picker v-model="singleSilenceStart" format="HH:mm" />
<a-time-picker v-model="singleSilenceEnd" format="HH:mm" />

const singleSilenceStart = ref('09:00')
const singleSilenceEnd = ref('18:00')

// 发送到后端
data.timeRanges = JSON.stringify([{
  weekdays: singleSilenceWeekdays.value,
  start: singleSilenceStart.value,  // ❌ 可能是时间对象，不是字符串
  end: singleSilenceEnd.value       // ❌ 可能是时间对象，不是字符串
}])
```

### 核心问题

**Arco Design 的 `a-time-picker` 组件**：
- `v-model` 绑定的是**时间对象**（Date 或 dayjs），不是字符串
- 初始值 `ref('09:00')` 是字符串
- 用户选择时间后，`singleSilenceStart.value` 会变成**时间对象**

**发送到后端的数据**：
```json
// 期望的格式
{
  "timeRanges": "[{\"weekdays\":[1,2,3,4,5],\"start\":\"09:00\",\"end\":\"20:00\"}]"
}

// 实际发送的格式（时间对象被 JSON.stringify 后）
{
  "timeRanges": "[{\"weekdays\":[1,2,3,4,5],\"start\":{},\"end\":{}}]"
}
```

**后端解析失败**：
```go
// isInTimeRanges 函数
currentTime := fmt.Sprintf("%02d:%02d", now.Hour(), now.Minute())  // "19:03"

// 比较逻辑
if currentTime >= r.Start && currentTime <= r.End {  // "19:03" >= "{}" ❌ 比较失败
    return true
}
```

**结果**：
- `r.Start` 和 `r.End` 是空对象 `{}`，不是 `"09:00"` 和 `"20:00"`
- 字符串比较失败，`isInTimeRanges` 返回 `false`
- 屏蔽规则不匹配，告警不被屏蔽

---

## ✅ 修复方案

### 修改内容

**文件**：`web/src/views/alert/ActiveEvents.vue`

**修复逻辑**：在发送到后端之前，将时间对象转换为 `HH:mm` 格式的字符串

### 1. 单条屏蔽修复

**修改位置**：第 569-577 行

**修复前**：
```typescript
if (singleSilenceType.value === 'fixed') {
  data.duration = silenceDuration.value
} else {
  data.timeRanges = JSON.stringify([{
    weekdays: singleSilenceWeekdays.value,
    start: singleSilenceStart.value,  // ❌ 可能是对象
    end: singleSilenceEnd.value       // ❌ 可能是对象
  }])

```

**修复后**：
```typescript
if (singleSilenceType.value === 'fixed') {
  data.duration = silenceDuration.value
} else {
  // 将时间对象转换为 HH:mm 格式的字符串
  const formatTime = (time: any): string => {
    if (typeof time === 'string') return time
    if (!time) return '00:00'
    // 如果是 Date 对象
    if (time instanceof Date) {
      const h = String(time.getHours()).padStart(2, '0')
      const m = String(time.getMinutes()).padStart(2, '0')
      return `${h}:${m}`
    }
    // 如果是 dayjs 对象
    if (time.format) {
      return time.format('HH:mm')
    }
    return '00:00'
  }

  data.timeRanges = JSON.stringify([{
    weekdays: singleSilenceWeekdays.value,
    start: formatTime(singleSilenceStart.value),  // ✅ 转换为字符串
    end: formatTime(singleSilenceEnd.value)       // ✅ 转换为字符串
  }])
}
```

## 🔧 formatTime 函数说明

**功能**：将时间对象转换为 `HH:mm` 格式的字符串

**支持的输入类型**：
1. **字符串**：直接返回（如 `"09:00"`）
2. **Date 对象**：提取小时和分钟，格式化为 `HH:mm`
3. **dayjs 对象**：使用 `format('HH:mm')` 方法
4. **null/undefined**：返回 `"00:00"`

**示例**：
```typescript
formatTime('09:00')                    // "09:00"
formatTime(new Date('2024-01-01 09:00')) // "09:00"
formatTime(dayjs('09:00', 'HH:mm'))    // "09:00"
formatTime(null)                       // "00:00"
```

## 🎉 总结

### 问题根因

- ✅ Arco Design 的 `a-time-picker` 组件返回时间对象，不是字符串
- ✅ 前端直接将时间对象发送到后端，导致 JSON 序列化后变成空对象 `{}`
- ✅ 后端字符串比较失败，屏蔽规则不匹配

### 修复方案

- ✅ 添加 `formatTime` 函数，将时间对象转换为 `HH:mm` 格式的字符串
- ✅ 在发送到后端之前，对 `start` 和 `end` 进行格式化
- ✅ 确保后端收到的是字符串格式的时间

### 修改文件

- ✅ `web/src/views/alert/ActiveEvents.vue` - 单条屏蔽和批量屏蔽
