# 周期性屏蔽规则不生效问题分析

## 🐛 问题描述

**现象**：
- 屏蔽规则：`主机-内存使用率过高`，周期性屏蔽，每天 09:00-20:00
- 告警时间：2026-05-07 19:03（在屏蔽时间段内）
- **问题**：告警仍然产生并推送通知

---

## 🔍 问题分析

### 可能原因 1：屏蔽规则的标签不匹配 ⭐ 最可能

**屏蔽规则匹配逻辑**（`matchSilenceRule` 函数）：
```go
func (e *EvalEngine) matchSilenceRule(rule *biz.AlertSilenceRule, event *biz.AlertEvent, now time.Time) bool {
    // 1. 检查告警等级
    if rule.Severity != event.Severity {
        return false
    }

    // 2. 检查规则名称
    if rule.RuleName != event.RuleName {
        return false
    }

    // 3. 检查标签（子集匹配）⭐ 关键点
    if !MatchLabels(event.Labels, rule.Labels) {
        return false
    }

    // 4. 检查时效
    if rule.Type == "periodic" {
        if isInTimeRanges(rule.TimeRanges, now) {
            return true
        }
    }

    return false
}
```

**标签匹配逻辑**（`MatchLabels` 函数）：
```go
func MatchLabels(eventLabelsJSON string, ruleLabelsJSON string) bool {
    eventMap := parseLabels(eventLabelsJSON)
    ruleMap := parseLabels(ruleLabelsJSON)

    // 规则标签必须是告警标签的子集
    for k, v := range ruleMap {
        if eventMap[k] != v {
            return false  // ⭐ 任何一个标签不匹配就返回 false
        }
    }
    return true
}
```

**问题**：
从图片中可以看到，告警的标签包含：
- `asset_group=星河核心机房`
- `category=host`
- `datasource=my-prom`
- `datasource_id=1`
- `datasource_type=prometheus`
- `ds_asset_group=星河核心服务力`
- `instance=172.16.38.129:9100`
- `job=iaas`
- `metric=memory`
- `ruleName=主机-内存使用率过高`
- `severity=critical`
- `team=ops`

**如果屏蔽规则的标签（`rule.Labels`）包含了告警中不存在的标签，或者标签值不匹配，就会导致匹配失败。**

---

### 可能原因 2：屏蔽规则未启用

**检查点**：
- 屏蔽规则的 `enabled` 字段是否为 `true`
- 从图片中看到规则状态是"启用"，所以这个不是问题

---

### 可能原因 3：时间范围配置错误

**检查点**：
- `TimeRanges` JSON 格式是否正确
- 星期几配置是否正确（1-7）
- 时间段配置是否正确（HH:mm 格式）

**从图片中看到**：
- 屏蔽时间：每天 09:00-20:00
- 告警时间：19:03（在屏蔽时间段内）
- 所以时间匹配应该是正确的

---

### 可能原因 4：缓存未更新

**检查点**：
- 屏蔽规则创建后，是否发布了重载事件
- Redis 缓存是否更新
- 本地缓存是否失效

**代码逻辑**：
```go
// 创建屏蔽规则后
if s.evalEngine != nil {
    if err := s.evalEngine.GetSilenceCache().PublishReloadEvent(c.Request.Context()); err != nil {
        c.Error(err)
    }
}
```

## 💡 解决方案

### 方案 1：检查屏蔽规则的标签配置 ⭐ 推荐

**问题**：
- 屏蔽规则保存的标签可能与实际告警的标签不完全匹配
- 例如：屏蔽规则保存了 `instance=172.16.38.129:9100`，但告警的 `instance` 可能略有不同

**解决方案**：
1. 查看屏蔽规则的详细配置，特别是 `Labels` 字段
2. 对比告警的 `Labels` 和屏蔽规则的 `Labels`
3. 确保屏蔽规则的标签是告警标签的子集

**验证方法**：
```sql
-- 查询屏蔽规则的标签
SELECT id, rule_name, severity, labels, time_ranges 
FROM alert_silence_rules 
WHERE rule_name = '主机-内存使用率过高';

-- 查询告警的标签
SELECT id, rule_name, severity, labels, fired_at 
FROM alert_events 
WHERE rule_name = '主机-内存使用率过高' 
ORDER BY fired_at DESC 
LIMIT 1;
```

**修复方法**：
- 如果标签不匹配，重新创建屏蔽规则
- 或者修改屏蔽规则，移除不匹配的标签

---

### 方案 2：优化标签匹配逻辑（代码修改）

**问题**：
- 当前逻辑要求屏蔽规则的所有标签都必须精确匹配
- 这可能导致一些合理的屏蔽规则无法生效

**解决方案**：
- 允许屏蔽规则的标签为空（匹配所有标签）
- 或者只匹配关键标签（如 `instance`、`job`）

**代码修改**：
```go
// 修改 MatchLabels 函数
func MatchLabels(eventLabelsJSON string, ruleLabelsJSON string) bool {
    eventMap := parseLabels(eventLabelsJSON)
    ruleMap := parseLabels(ruleLabelsJSON)

    // ✅ 如果规则标签为空，匹配所有告警
    if len(ruleMap) == 0 {
        return true
    }

    // 规则标签必须是告警标签的子集
    for k, v := range ruleMap {
        if eventMap[k] != v {
            return false
        }
    }
    return true
}
```

### 方案 3：添加调试日志

**目的**：
- 了解屏蔽规则为什么没有匹配成功
- 定位具体是哪个条件不满足

**代码修改**：
```go
func (e *EvalEngine) matchSilenceRule(rule *biz.AlertSilenceRule, event *biz.AlertEvent, now time.Time) bool {
    // 1. 检查告警等级
    if rule.Severity != event.Severity {
        appLogger.Debug("屏蔽规则不匹配：告警等级不同",
            zap.String("ruleSeverity", rule.Severity),
            zap.String("eventSeverity", event.Severity))
        return false
    }

    // 2. 检查规则名称
    if rule.RuleName != event.RuleName {
        appLogger.Debug("屏蔽规则不匹配：规则名称不同",
            zap.String("ruleRuleName", rule.RuleName),
            zap.String("eventRuleName", event.RuleName))
        return false
    }

    // 3. 检查标签（子集匹配）
    if !MatchLabels(event.Labels, rule.Labels) {
        appLogger.Debug("屏蔽规则不匹配：标签不匹配",
            zap.Uint("silenceRuleID", rule.ID),
            zap.String("ruleLabels", rule.Labels),
            zap.String("eventLabels", event.Labels))
        return false
    }

    // 4. 检查时效
    if rule.Type == "fixed" {
        if rule.SilenceUntil != nil && now.Before(*rule.SilenceUntil) {
            appLogger.Debug("屏蔽规则匹配成功（固定时长）",
                zap.Uint("silenceRuleID", rule.ID))
            return true
        }
    } else if rule.Type == "periodic" {
        if isInTimeRanges(rule.TimeRanges, now) {
            appLogger.Debug("屏蔽规则匹配成功（周期性）",
                zap.Uint("silenceRuleID", rule.ID),
                zap.String("timeRanges", rule.TimeRanges))
            return true
        } else {
            appLogger.Debug("屏蔽规则不匹配：不在时间窗口内",
                zap.Uint("silenceRuleID", rule.ID),
                zap.String("timeRanges", rule.TimeRanges),
                zap.Time("now", now))
        }
    }

    return false
}
```
