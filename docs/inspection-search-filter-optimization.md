# 智能巡检搜索与筛选功能优化说明

## 优化概述

在智能巡检系统的两个页面新增了巡检组、巡检项模糊搜索能力，并在任务调度页面额外新增了状态筛选条件，提升用户快速定位和筛选巡检数据的效率。

**页面位置**：`InspectionRecords.vue` - 详情弹窗的"执行明细"标签页

**优化前**：
- ✅ 执行状态筛选（成功/失败）
- ✅ 断言结果筛选（通过/失败/跳过）

**优化后**：
- ✅ 执行状态筛选（成功/失败）
- ✅ 断言结果筛选（通过/失败/跳过）
- ➕ **巡检组名称模糊搜索**
- ➕ **巡检项名称模糊搜索**

**功能特性**：
- 🔍 实时搜索，输入即筛选
- 🔤 大小写不敏感
- 🌐 支持中文、英文、数字
- 🔄 支持多条件组合筛选
- 🧹 一键重置所有筛选条件

**优化后**：

#### 2. 模糊搜索实现

```typescript
// 巡检组名称模糊搜索（大小写不敏感）
if (filter.groupName) {
  filtered = filtered.filter(item =>
    item.groupName?.toLowerCase().includes(filter.groupName.toLowerCase())
  )
}

// 巡检项名称模糊搜索（大小写不敏感）
if (filter.itemName) {
  filtered = filtered.filter(item =>
    item.itemName?.toLowerCase().includes(filter.itemName.toLowerCase())
  )
}
```

#### 3. 状态筛选实现

```typescript
// 执行状态筛选
if (filter.status) {
  filtered = filtered.filter(item => item.status === filter.status)
}

// 断言结果筛选
if (filter.assertionResult) {
  filtered = filtered.filter(item => item.assertionResult === filter.assertionResult)
}
```

```typescript
const resetFilter = () => {
  filter.groupName = ''
```

## 使用说明

### 场景 1：巡检执行记录详情页面

**步骤**：
1. 点击巡检执行记录的"查看详情"按钮
2. 切换到"执行明细"标签页
3. 使用筛选条件：
   - 在"搜索巡检组名称"输入框输入关键词（如"数据库"）
   - 在"搜索巡检项名称"输入框输入关键词（如"CPU"）
   - 选择"执行状态"（成功/失败）
   - 选择"断言结果"（通过/失败/跳过）
4. 点击"重置"按钮清空所有筛选条件

**示例**：
- 搜索"CPU"：快速筛选出所有 CPU 相关的巡检项
- 搜索"数据库" + 状态"失败"：快速定位数据库巡检组中失败的项
- 搜索"内存" + 断言"fail"：快速定位内存相关且断言失败的项

### 场景 2：任务调度页面 - 立即运行结果

**步骤**：
1. 在任务调度页面点击"立即运行"按钮
2. 等待执行完成，查看结果抽屉
3. 使用筛选条件：
   - 在"搜索巡检组名称"输入框输入关键词
   - 在"搜索巡检项名称"输入框输入关键词
   - 选择"执行状态"（成功/失败）
   - 选择"断言状态"（通过/失败/跳过）
4. 查看筛选结果统计（已筛选 X / 总数 Y 条）
5. 点击"重置"按钮清空所有筛选条件

**示例**：
- 状态"失败"：快速查看所有失败的巡检项
- 断言"fail"：快速查看所有断言失败的巡检项
- 搜索"网络" + 状态"失败"：快速定位网络相关的失败项
- 搜索"磁盘" + 断言"fail"：快速定位磁盘相关且断言失败的项

## 功能特性详解

### 1. 模糊搜索

**特性**：
- 大小写不敏感：输入"cpu"可以匹配"CPU使用率"
- 部分匹配：输入"数据"可以匹配"数据库巡检"、"数据采集"
- 支持中英文：输入"disk"可以匹配"磁盘使用率"

**示例**：
```
输入: "cpu"
匹配: "CPU使用率"、"cpu_usage"、"服务器CPU监控"

输入: "数据"
匹配: "数据库巡检"、"数据采集"、"数据同步检查"

输入: "mem"
匹配: "Memory使用率"、"内存(memory)检查"
```

### 2. 多条件组合筛选

**逻辑**：所有条件必须同时满足（AND 关系）

**示例**：
```
条件 1: 巡检组名称 = "数据库"
条件 2: 执行状态 = "失败"
条件 3: 断言结果 = "fail"

结果: 只显示"数据库"巡检组中，执行状态为"失败"且断言结果为"fail"的巡检项
```

### 3. 实时筛选

**特性**：
- 输入框：输入即筛选（`@input` 事件）
- 下拉框：选择即筛选（`@change` 事件）
- 无需点击"查询"按钮

### 4. 筛选结果统计

**位置**：任务调度页面 - 立即运行结果弹窗

**显示**：
```
[已筛选 10 / 50 条]
```

**说明**：
- 10：当前筛选后的结果数量
- 50：总数据量
- 仅在筛选条件生效时显示

## 常见问题

### Q1: 搜索不区分大小写吗？

**A**: 是的，搜索不区分大小写。输入"cpu"可以匹配"CPU"、"Cpu"、"cpu"。

### Q2: 可以同时搜索多个关键词吗？

**A**: 当前版本不支持。如需搜索多个关键词，建议分别搜索或使用组合筛选。

### Q3: 筛选条件是 AND 还是 OR 关系？

**A**: 所有筛选条件是 AND 关系，即必须同时满足所有条件。

### Q4: 筛选后的数据可以导出吗？

**A**: 当前版本筛选仅影响前端显示，不影响导出功能。导出仍然是全量数据。

### Q5: 数据量很大时会卡顿吗？

**A**: 当前场景数据量通常在几十到几百条，不会卡顿。如果数据量超过 1000 条，建议联系管理员优化为后端筛选。
