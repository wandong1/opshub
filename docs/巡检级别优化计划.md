# 巡检级别与风险等级优化计划

## 📋 需求概述

在智能巡检系统中，为每个巡检项增加**巡检级别**和**风险等级**字段（高/中/低），并在以下场景中完整展示这些信息：

1. 巡检管理 - 测试执行结果
2. 任务调度管理 - 执行结果展示
3. 执行记录 - 历史记录详情
4. 导出报告 - Excel 报告

## 🎯 优化目标

- 为巡检项增加级别分类，便于优先级管理和风险评估
- 在所有执行结果中完整展示巡检项的关键信息
- 提升巡检报告的可读性和决策价值
- 保持数据一致性，避免字段遗漏

## 🔧 详细实施步骤

### 第一步：数据库迁移（自动迁移）

```sql
-- 巡检项表增加字段
ALTER TABLE inspection_items 
ADD COLUMN inspection_level VARCHAR(20) DEFAULT 'medium' COMMENT '巡检级别: high/medium/low',
ADD COLUMN risk_level VARCHAR(20) DEFAULT 'medium' COMMENT '风险等级: high/medium/low';

-- 执行明细表增加字段
ALTER TABLE inspection_execution_details
ADD COLUMN inspection_level VARCHAR(20) DEFAULT '' COMMENT '巡检级别',
ADD COLUMN risk_level VARCHAR(20) DEFAULT '' COMMENT '风险等级';
```

**辅助方法：**

```typescript
const getLevelText = (level: string) => {
  const map: Record<string, string> = {
    high: '高',
    medium: '中',
    low: '低'
  }
  return map[level] || '中'
}

const getLevelColor = (level: string) => {
  const map: Record<string, string> = {
    high: 'red',
    medium: 'orange',
    low: 'green'
  }
  return map[level] || 'orange'
}

const getRiskColor = (level: string) => {
  const map: Record<string, string> = {
    high: 'red',
    medium: 'orangered',
    low: 'blue'
  }
  return map[level] || 'orangered'
}
```
