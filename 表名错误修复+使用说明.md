# 表名错误修复 + 使用说明

## 修复的问题

### 1. ✅ 表名错误（sys_users → sys_user）

**错误日志**：
```
Error 1146 (42S02): Table 'opshub.sys_users' doesn't exist
```

**原因**：`group_service.go` 第 223 行使用了错误的表名 `sys_users`（多了个 s）

**修复**：
```go
// 修改前 ❌
db.WithContext(ctx).Table("sys_users").Select("phone")...

// 修改后 ✅
db.WithContext(ctx).Table("sys_user").Select("phone")...
```

**文件**：`internal/service/alert/group_service.go`

---

### 2. ✅ 字段名错误（name → real_name）

**错误日志**：
```
Error 1054 (42S22): Unknown column 'name' in 'field list'
```

**原因**：`notify_service.go` 使用了 `name` 字段，但表中是 `real_name`

**修复**：
```go
// 修改前 ❌
Select("email, name")

// 修改后 ✅
Select("email, real_name")
```

**文件**：`internal/service/alert/notify_service.go`

## 邮件发送逻辑说明

### 日志："邮件通道未指定接收用户，跳过发送"

这是**正常行为**，不是错误！

**原因**：告警订阅中没有选择接收用户

**解决方法**：

#### 步骤 1：配置用户邮箱

进入"系统管理" → "用户管理" → 编辑用户：

| 字段 | 说明 | 示例 |
|------|------|------|
| 手机号 | 必填，用于关联 | 18182294500 |
| 邮箱 | 必填，接收邮件 | user@example.com |
| 真实姓名 | 可选 | 张三 |

#### 步骤 2：配置告警订阅

进入"告警治理" → "告警订阅" → 编辑订阅：

1. **选择通知通道**：勾选邮件通道
2. **选择接收用户**：勾选需要接收邮件的用户（必须！）
3. 保存

**重要**：如果不选择接收用户，邮件通道会跳过发送。

#### 步骤 3：查看日志

```bash
tail -f logs/app.log | grep "邮件"
```

**正常日志**（选择了用户）：
```
INFO    准备发送邮件    {"smtpHost": "smtp.qq.com", "smtpPort": 465, "phones": ["18182294500"]}
INFO    找到接收邮箱    {"emails": ["user@example.com"]}
INFO    邮件发送成功    {"emails": ["user@example.com"]}
```

**跳过日志**（未选择用户）：
```
INFO    邮件通道未指定接收用户，跳过发送
```

## 常见问题

### Q1：日志显示"邮件通道未指定接收用户"

**原因**：告警订阅中没有选择接收用户

**解决**：
1. 进入"告警治理" → "告警订阅"
2. 编辑订阅，勾选接收用户
3. 保存

### Q2：日志显示"未找到有效的用户邮箱"

**原因**：选择的用户没有配置邮箱

**解决**：
1. 进入"系统管理" → "用户管理"
2. 编辑用户，填写邮箱地址
3. 保存

### Q3：日志显示"SMTP 认证失败"

**原因**：密码错误或未使用授权码

**解决**：
- QQ 邮箱：使用"授权码"，不是 QQ 密码
- 获取方式：QQ邮箱 → 设置 → 账户 → POP3/SMTP/IMAP → 开启服务 → 生成授权码

### Q4：邮件发送成功但没收到

**可能原因**：
1. 邮件在垃圾箱
2. 邮箱地址填写错误
3. SMTP 服务器限制

**排查**：
1. 检查垃圾箱
2. 检查用户邮箱地址是否正确
3. 查看完整日志确认发送状态
